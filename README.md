## Технології та підхід

- **Unity**
- **Zenject** (DI / інʼєкція залежностей, інсталяція сервісів та фабрик)
- **2D sprites**
  - Використовуються **atlases** та **slices** (нарізка спрайтів)
- **ScriptableObject**
  - Конфіги та параметри ігрових сутностей зберігаються в ассетах, що спрощує баланс та розширення контенту
- **Dotween**
  - Анімації руху
- **TrailRenderer**
  - Анімації хвоста у куль та колес
- **Gpu instancing**
  - зменшення батчей
  
## Структура проєкту

- **`Assets/GameResources/`**
  - **`Game/`**
    - Ігрові ресурси (префаби, графіка, тощо) які використовуються під час гри
  - **`Scenes/`**
    - Сцени Unity
  - **`Ui/`**
    - Ресурси та префаби для інтерфейса

- **`Assets/Scripts/`**
  - **`ConfigData/`**
    - Моделі/структури даних, що описують конфіги та їхні залежності
  - **`Core/`**
    - Базова інфраструктура: загальні сервіси/утиліти/контракти. А також інсталери
  - **`GameServices/`**
    - Ігрові сервіси та контролери
  - **`Preloader/`**
    - контролер на сцені прелоадер
  - **`UI/`**
    - скрипти для UI

- **`Assets/Resources/GameConfig`**
  - Конфіг з даними для гри: рівні,вороги, машини гравця і т.д.
- **`Assets/Plugins/`**
  - Плагіни, зокрема **Zenject** та **DOTween** (Demigiant)

## В планах

- **Розбити конфіг на складові** (більш модульна структура ScriptableObject-конфігів):
  - Окремий конфіг для **рівнів**
  - Окремий конфіг для **машин гравця**
  - Окремий конфіг для **ігрових персонажів** (вороги/союзники/нейтральні)
- **підв'язати усі прості матеріали до загальної текстури з кольорами**
  - матеріали для перешвод, куль та інші використовують окремі кольори замість кольору з текстури.
- **flash ефект при попаданні має бути просто зміною параметра шейдера чи material_property_block**
 - наразі замінюється просто на інший матеріал
- **зробити геймплей більш різноманітним**
  - переміщення гравця не тільки по прямій
  - більше видів ворогів, турелей, куль та машин
- **new input system**
- **інтегрувати пауз систему у `GameInstance`**
